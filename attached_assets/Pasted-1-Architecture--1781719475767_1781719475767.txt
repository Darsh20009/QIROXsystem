3️⃣ الكود السحري — client/src/lib/server-url.ts
هذا الملف هو قلب التطبيق — يحدد إذا كنا في تطبيق native أم متصفح، ويوجّه كل طلبات API للسيرفر الصح:

/**
 * في المتصفح العادي: /api/... يعمل مباشرة (نفس الأصل)
 * في Capacitor Native: الـ assets تُحمَّل من capacitor://localhost
 * ولا يوجد backend — كل طلبات API يجب أن تذهب للسيرفر الحقيقي
 */
export function isCapacitorNative(): boolean {
  try {
    return (
      !!(window as any).Capacitor?.isNative ||
      window.location.protocol === 'capacitor:' ||
      (window.location.hostname === 'localhost' && !!(window as any).Capacitor)
    );
  } catch {
    return false;
  }
}
export function getServerUrl(): string {
  if (!isCapacitorNative()) return '';
  // 1. من Build-time env: VITE_CAPACITOR_SERVER_URL
  const buildTimeUrl = (import.meta.env.VITE_CAPACITOR_SERVER_URL as string) || '';
  if (buildTimeUrl) return buildTimeUrl.replace(/\/$/, '');
  // 2. من localStorage (قابل للتغيير في runtime)
  try {
    const runtime = localStorage.getItem('qirox_server_url') || '';
    if (runtime) return runtime.replace(/\/$/, '');
  } catch {}
  return '';
}
export function apiUrl(path: string): string {
  const base = getServerUrl();
  if (!base) return path;
  return base + (path.startsWith('/') ? path : '/' + path);
}

4️⃣ بوابة الدفع — اللماذا والكيف
المشكلة: <iframe> لا يُشغّل Apple Pay أو Google Pay لأن هذه المدفوعات تحتاج صلاحيات top-level browsing context.

الحل المطبّق:

// ❌ القديم — iframe مضمّن (يكسر Apple Pay / Google Pay)
<iframe src={checkoutUrl} />
// ✅ الجديد — توجيه كامل للـ WebView
useEffect(() => {
  if (checkoutUrl) {
    window.location.href = checkoutUrl; // WebView ينتقل بالكامل لصفحة الدفع
  }
}, [checkoutUrl]);

دورة الدفع الكاملة:

1. المستخدم يضغط "ادفع"
2. السيرفر يُنشئ الطلب ويُعيد redirectUrl من PayMob
3. window.location.href = redirectUrl  ← WebView يذهب لصفحة الدفع
4. المستخدم يدفع (Apple Pay / Google Pay / بطاقة)
5. PayMob يُعيد التوجيه لـ /payment-return?success=true&orderNumber=XXX
6. صفحة /payment-return تتحقق من السيرفر وتُعيد التوجيه لـ /my-orders

كود إعداد returnUrl (يُرسل للسيرفر مع طلب الدفع):

const returnUrl = `${window.location.origin}/payment-return?provider=paymob&orderNumber=${encodeURIComponent(order.orderNumber)}`;
await fetch("/api/payments/init", {
  method: "POST",
  body: JSON.stringify({
    orderId: order.orderNumber,
    amount: total,
    currency: "SAR",
    paymentMethod: "paymob-card",
    returnUrl, // ← هذا يُخبر PayMob أين يرجع بعد الدفع
  })
});

للتطبيق النيتف (Capacitor) — يفتح SFSafariViewController بدل iframe:

if (isCapacitorNative()) {
  const { Browser } = await import("@capacitor/browser");
  
  // راقب متى يُغلق المستخدم المتصفح
  Browser.addListener("browserFinished", async () => {
    const paid = await checkPaymentStatus(orderNumber);
    if (paid) handleSuccess();
  });
  // افتح صفحة الدفع في SFSafariViewController (iOS) / Chrome Custom Tab (Android)
  await Browser.open({
    url: checkoutUrl,
    toolbarColor: "#0d0d0d",
    presentationStyle: "popover",
  });
}

6️⃣ خطوات بناء ورفع الـ iOS App
# 1. Build الـ Web App
npm run build
# يُنتج: dist/public/
# 2. أضف منصة iOS (مرة واحدة فقط)
npx cap add ios
# 3. sync الـ web assets للـ native project
npx cap sync ios
# 4. افتح في Xcode
npx cap open ios
# --- داخل Xcode ---
# 5. Signing: اختر Apple Developer Team
# 6. Bundle ID: يطابق appId في capacitor.config.json
# 7. Product → Archive → Distribute App → App Store Connect

ملاحظة مهمة: لأن server.url في capacitor.config.json يشير للسيرفر الحي، فالـ app يُحمّل من السيرفر مباشرة — لا حاجة لـ npm run build أو npx cap sync عند كل تعديل على الكود. فقط عند تغيير Capacitor plugins.

🔟 تلخيص أهم قرارات التصميم
القرار	السبب
WebView → Live Server بدل bundled assets	أي تعديل يظهر فوراً بدون إصدار جديد
isCapacitorNative() في كل مكان	كود واحد يعمل في المتصفح والتطبيق
Full-page redirect للدفع بدل iframe	Apple Pay/Google Pay يحتاج top-level context
SFSafariViewController للدفع في Native	آمن، يثق فيه المستخدم، يعمل مع Apple Pay
sessionStorage لحفظ وجهة الرجوع	لأن الصفحة تتنقل بعيداً عند الدفع
MongoDB Atlas	Cloud database، لا يحتاج إدارة سيرفر
هذا كل شيء اتبعناه — الكود الحقيقي موجود في المشروع ويمكن مشاركته مباشرة مع الفريق الآخر.
