# تطوير نماذج الأعمال - ملفات شرح ومميزات وطريقة استخدام

## What & Why
تطوير صفحة إدارة الخدمات (AdminServices) لتمكين الموظف من إرفاق ملفات شرح (PDF + فيديو) بدون قيد حجم، إضافة رابط المنصة، المميزات التفصيلية، وطريقة الاستخدام لكل نموذج عمل. حالياً الصفحة تدعم فقط صور المحفظة ورابط واحد وقائمة مميزات نصية بسيطة.

## Done looks like
- الموظف يستطيع رفع ملفات شرح (PDF وفيديو) بدون قيد على الحجم لكل خدمة/نموذج عمل
- حقل رابط المنصة (Platform URL) واضح ومنفصل عن رابط المحفظة
- حقل طريقة الاستخدام (Usage Instructions) كحقل نصي طويل يدعم خطوات مرقمة
- المميزات تظهر بشكل واضح (القائمة الحالية تبقى مع تحسينها)
- في واجهة العميل: الفيديوهات تُعرض مباشرة داخل الصفحة (مشغل فيديو مدمج) مع إمكانية التحميل، ملفات PDF تفتح أو تُحمّل
- رفع الحد الأقصى لحجم الملف من 20MB الحالي (أو إنشاء endpoint رفع منفصل بدون حد حجم لهذه الملفات)

## Out of scope
- تسليم المشاريع للعملاء (مهمة منفصلة)
- تغيير تصميم صفحة الخدمات العامة بالكامل

## Tasks
1. **تحديث نموذج الخدمة (Schema)** — إضافة حقول: portfolioFiles (مصفوفة كائنات تحتوي url, type, name), platformUrl, usageInstructions إلى serviceSchema في models.ts.
2. **إنشاء endpoint رفع ملفات كبيرة** — إضافة POST /api/upload/large بدون قيد حجم (أو حد عالي جداً مثل 500MB) يقبل PDF وفيديو، مع حماية بالمصادقة.
3. **تطوير واجهة AdminServices** — إضافة حقول: رفع ملفات الشرح (PDF+فيديو)، رابط المنصة، طريقة الاستخدام (textarea)، مع عرض الملفات المرفقة وإمكانية حذفها.
4. **تطوير عرض العميل** — في الصفحات التي يرى فيها العميل تفاصيل الخدمة: عرض الفيديو مباشرة بمشغل مدمج + زر تحميل، عرض ملفات PDF كروابط تحميل/فتح، وعرض طريقة الاستخدام.
